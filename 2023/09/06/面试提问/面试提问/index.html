项目中遇到的难点

在开发公司低代码数据可视化平台时，因为我们开发了许多的组件，在打开配置页面的时候第一次会加载很久，我就把每一个组件的.vue文件单独提出来，使用webpack打包成js文件上传到服务器，进入页面的时候就使用异步组件的方式，用网络请求把存储在服务器上的组件请求下来，用loadjs库加载出来再转换为vue组件，就加快了页面首次进入的时间。

在开发公司的大屏项目时，做大屏项目时因为要在不同的设备上查看效果、开发。很有可能因为分辨率的问题导致在每个人的电脑上展示页面不一样。后来我的解决方案是开发一个全屏容器组件，给定整体的分辨率，然后计算出长宽比例，当组件的大小发生变化时，使里面的内容根据长宽比例缩放，动态更改css的transform的scale属性，这样就可以让容器不会变形。

在开发公司的项目时，有许多的子项目，在每次部署上线时，有的时候项目会打包很慢，就把公司的打包配置进行了优化。

  1. 使用了SourceMap（源代码映射），当构建后代码出错会帮我们找到映射后源代码出错的位置，从而让浏览器提示源代码文件出错位置，让我们更快的找到错误根源。对打包比较耗时的项目开启多线程打包，
  2. 使用thread-loader。为了提高首屏加载速度，在打包的时候就开启gzip压缩，可以使用compression-webpack-plugin插件。
  3. 使用code Split分割代码，减少首页加载js，code split可以将打包的文件进行分割，生成多个js文件。也实现了按需加载，需要哪个文件就加载哪个文件
  4. 使用 Preload / Prefetch 对代码进行提前加载，等未来需要使用时就能直接使用，从而用户体验更好。
  5. 使用 OneOf 让资源文件一旦被某个 loader处理了，就不会继续遍历了，打包速度更快。
  6. 使用 HotModuleReplacement 让开发时只重新编译打包更新变化了的代码，不变的代码使用缓存，从而使更新速度更快。

let const var 有什么区别

  • var声明变量存在变量提升，let和const不存在变量提升
  • let和const只能在块作用域里访问
  • 同一作用域下let和const不能声明同名变量，而var可以
  • const定义常量，而且不能修改，但是在定义的对象时对象属性值可以改变

WeakMap和WeakSet可以说一说吗

WeakMap是一种键值对的集合，类似于Map。不过，WeakMap与Map有几个重要的区别：

  1. 在WeakMap中，只有对象可以作为键。换句话说，我们不能使用基本类型（如数字，字符串，布尔值等）作为WeakMap的键。
  2. WeakMap的键是弱引用的。这意味着，如果一个对象只被WeakMap引用，那么这个对象可以被垃圾回收（GC）。当这个对象被垃圾回收后，它对应的键值对也会从WeakMap中自动移除。
  3. WeakMap不可遍历，也就是说，我们不能使用像for...of这样的循环来遍历WeakMap。

由于这些特性，WeakMap在处理内存泄漏问题和管理对象私有数据等场景中有着显著的优势。

WeakSet也是一种集合，类似于Set。WeakSet与Set的主要区别包括：

  1. 在WeakSet中，只有对象可以作为值。也就是说，我们不能将基本类型（如数字，字符串，布尔值等）添加到WeakSet中。
  2. WeakSet中的对象是弱引用的。如果一个对象只被WeakSet引用，那么这个对象可以被垃圾回收。当这个对象被垃圾回收后，它会自动从WeakSet中移除。
  3. WeakSet不可遍历，也就是说，我们不能使用像for...of这样的循环来遍历WeakSet。

WeakSet在处理对象的唯一性、内存泄漏等问题上有其独特的应用。
