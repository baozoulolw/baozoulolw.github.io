面试提问

自我介绍

面试官们好，我叫王浩然，今年24岁，来自重庆，来应聘的是贵公司的前端开发岗位。我本人比较擅长vue开发，可以熟练使用vue2，有相对丰富对项目经验，对vue3也有开发经验。可以熟练使用js、css、html三件套，对typescript和less也有使用经验。对地图和大屏的开发也比较熟悉，使用过echarts、openlayer、cesium等前端框架。同时我对后端java技术栈也有一定了解，在上家公司担任过一段时间的前端开发小组长，带领公司的实习生开发。开发过多个大屏和后台管理项目，小程序和app开发也有涉及，希望能胜任此职位。

项目中遇到的难点

在开发公司低代码数据可视化平台时，因为我们开发了许多的组件，在打开配置页面的时候第一次会加载很久，我就把每一个组件的.vue文件单独提出来，使用webpack打包成js文件上传到服务器，进入页面的时候就使用异步组件的方式，用网络请求把存储在服务器上的组件请求下来，用loadjs库加载出来再转换为vue组件，就加快了页面首次进入的时间。

在开发公司的大屏项目时，做大屏项目时因为要在不同的设备上查看效果、开发。很有可能因为分辨率的问题导致在每个人的电脑上展示页面不一样。后来我的解决方案是开发一个全屏容器组件，给定整体的分辨率，然后计算出长宽比例，当组件的大小发生变化时，使里面的内容根据长宽比例缩放，动态更改css的transform的scale属性，这样就可以让容器不会变形。

在开发公司的项目时，有许多的子项目，在每次部署上线时，有的时候项目会打包很慢，就把公司的打包配置进行了优化。
使用了SourceMap（源代码映射），当构建后代码出错会帮我们找到映射后源代码出错的位置，从而让浏览器提示源代码文件出错位置，让我们更快的找到错误根源。对打包比较耗时的项目开启多线程打包，
2. 使用thread-loader。为了提高首屏加载速度，在打包的时候就开启gzip压缩，可以使用compression-webpack-plugin插件。
3. 使用code Split分割代码，减少首页加载js，code split可以将打包的文件进行分割，生成多个js文件。也实现了按需加载，需要哪个文件就加载哪个文件
4. 使用 Preload / Prefetch 对代码进行提前加载，等未来需要使用时就能直接使用，从而用户体验更好。
5. 使用 OneOf 让资源文件一旦被某个 loader处理了，就不会继续遍历了，打包速度更快。
6. 使用 HotModuleReplacement 让开发时只重新编译打包更新变化了的代码，不变的代码使用缓存，从而使更新速度更快。
